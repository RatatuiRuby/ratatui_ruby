# Prompt for Developer Agent: Phase 2, Step 2.1

You are a Senior Ruby Developer. Your task is to create the Ruby-side `Frame` class definition for documentation and type-checking purposes. This implements **Phase 2, Step 2.1** of the [Frame Migration Plan](../../../doc/contributors/frame_migration.md).

## Context

Previous agents have implemented the Rust backend:
- `ext/ratatui_ruby/src/frame.rs`: `RubyFrame` struct with `area()` and `render_widget()` methods.
- `ext/ratatui_ruby/src/lib.rs`: Updated `draw` to accept a block and yield a `Frame`.

The actual implementation lives in Rust. Your job is to create the **Ruby stub** for RDoc documentation and the **RBS type signature** for type checking. This pattern is used throughout RatatuiRuby where Rust-backed classes need Ruby-side documentation.

## Task Specs

### 1. Create Ruby Class Definition

**File**: `lib/ratatui_ruby/frame.rb`

- Define `RatatuiRuby::Frame` class.
- Add RDoc documentation following the project's **documentation style guide** at `doc/contributors/documentation_style.md`:
    - Use **Context-Problem-Solution** narrative structure.
    - Explain what the Frame is, the problem it solves, and how to use it.
    - Include `=== Examples` sections showing basic usage and usage with `Layout.split`.
    - Use `<tt>` tags for code literals, NOT backticks.
    - Do NOT use YARD tags (`@param`, `@return`, etc.). Use standard RDoc formats.
- Document methods:
    - `#area`: Returns the full terminal area as a `Rect`.
    - `#render_widget(widget, area)`: Renders a widget at the specified area.
- Method bodies can be empty or contain a comment like `# Implemented in Rust`.
- **Pattern Reference**: See `lib/ratatui_ruby/schema/rect.rb` for how other Rust-backed classes are documented.

### 2. Create RBS Type Signature

**File**: `sig/ratatui_ruby/frame.rbs`

- Define type signatures for:
    - `def area: () -> Rect`
    - `def render_widget: (widget widget, Rect area) -> nil`
- **Pattern Reference**: See `sig/ratatui_ruby/schema/rect.rbs` for RBS conventions.
- Note: `widget` is a type alias defined elsewhere in the RBS files.

### 3. Require the File

**File**: `lib/ratatui_ruby.rb`

- Add `require_relative "ratatui_ruby/frame"` in the appropriate location with other requires.

## References

- `sig/ratatui_ruby/schema/rect.rbs`: RBS typing conventions.

## Definition of Done
- `lib/ratatui_ruby/frame.rb` exists with complete RDoc documentation following project style.
- `sig/ratatui_ruby/frame.rbs` exists with correct type signatures.
- `lib/ratatui_ruby.rb` requires the new file.
- Running `bundle exec rake rdoc` includes Frame in the generated documentation.
